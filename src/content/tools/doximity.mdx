## Overview

Doximity is one of the largest verified clinician networks in the United States, and its AI tools are embedded directly within that network. The suite has grown well beyond letter drafting: Doximity GPT handles drafting and clinical queries, Doximity Scribe provides ambient documentation and dictation, and Doximity Ask answers clinical questions, with physician-led review of AI answers (PeerCheck) rolling out in 2026. For physicians already using Doximity for faxing, messaging, and professional networking, these tools are a natural extension, and all of them are free for verified U.S. physicians, NPs, PAs, and medical students.

Doximity publishes a member-facing Business Associate Agreement, and enterprise BAAs are available for organizational deployments. The platform states that users can include PHI in prompts within designated secure features, with data encrypted both in transit and at rest.

The key detail: not every Doximity feature carries BAA coverage. Understanding which tools are designated as HIPAA-secure versus which are general-purpose is essential before entering patient data.

For clinicians who already live inside the Doximity ecosystem, the AI tools are conveniently accessible. For those evaluating Doximity purely as an AI documentation tool, the question is whether its drafting and communication capabilities justify the platform commitment.

Doximity's primary advantage is convenience: if you already use it for faxing and messaging, adding AI-assisted letter drafting requires no new software, no new login, and no new procurement process. That low friction is both a benefit and a risk: it makes it easy to start using AI features without fully understanding which ones carry BAA coverage.

## Weaknesses

Be aware of these limitations before relying on Doximity for PHI-containing workflows:

- **Not all features are HIPAA-designated**: Doximity offers both secure and general-purpose tools. Using the wrong mode for PHI-containing work creates compliance risk.
- **Feature boundaries can be unclear**: knowing exactly which tool mode you are in requires attention. There is no universal "safe mode" that covers everything.
- **BAA retention language**: the BAA describes data retention when return or destruction is not feasible, which may not align with your organization's data minimization expectations
- **Chart volume risk**: avoid uploading entire charts when only a specific section is needed. More data in the prompt means more data subject to the platform's retention and processing terms.
- **Limited clinical reasoning depth**: Doximity GPT is useful for drafting and communication but is not positioned as a clinical decision support tool
- **No direct EHR integration**: Scribe generates the note, but getting it into the chart is still a copy-paste step; enterprise ambient tools write into the EHR directly

<Callout type="pitfall">
  Before entering PHI into any Doximity tool, confirm you are using a feature explicitly designated as HIPAA-secure. The BAA does not blanket-cover every feature on the platform. When in doubt, check Doximity's current documentation or ask your compliance team.
</Callout>

## HIPAA & Privacy

<Callout type="hipaa">
  Doximity publishes a member-facing BAA and describes enterprise BAAs for organizational deployments. Data is stated to be encrypted in transit and at rest. However, BAA coverage applies only to features explicitly designated as HIPAA-secure, not to every tool within the Doximity ecosystem. Confirm which features carry coverage before entering patient data.
</Callout>

Key compliance details:

- **Member-facing BAA**: published and available for individual clinicians to review and accept
- **Enterprise BAAs**: available for organizational deployments with additional terms
- **Encryption**: data encrypted in transit and at rest
- **Retention**: BAA describes retention when return or destruction of data is not feasible; review these terms against your organization's policies
- **Scope limitations**: only designated secure features are covered under the BAA

For individual physicians: the member-facing BAA is a meaningful step, but verify that the specific features you are using fall within its scope. For organizations: pursue an enterprise BAA that explicitly names covered products and features.

Bottom line: Doximity's BAA is real and accessible, which puts it ahead of many competitors. The critical step is confirming feature-level coverage so that you are using the right tool within the platform for PHI-containing work.

If you are already a Doximity member, the AI tools are worth exploring for communication and drafting tasks. Just take five minutes to understand which features are covered before you start entering patient information.
