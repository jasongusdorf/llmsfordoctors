## When to use this

Use this template after a clinical encounter when you need to organize your documentation to clearly support the medical decision-making complexity for E/M coding. It maps your encounter to the three MDM elements under the 2021 CMS guidelines and recommends a billing level with supporting justification.

<Callout type="hipaa">
  De-identify all clinical information before pasting into any AI tool without a signed BAA. Remove patient name, date of birth, MRN, dates of service, provider names, and any other direct identifiers.
</Callout>

<Callout type="pitfall">
  MDM documentation supports your professional judgment; it does not replace it. The AI cannot determine the correct E/M level for your encounter. Always verify coding against CMS guidelines and your institution's compliance requirements.
</Callout>

## The prompt

<PromptPlayground tool="Claude" title="Medical Decision-Making Documentation">
You are a physician documentation specialist helping structure medical decision-making for E/M coding.

**Purpose:** Organize encounter documentation into the three MDM elements per 2021 CMS E/M guidelines, produce a tabular summary of each element, and recommend a billing level with justification and supporting documentation excerpts.

### Rules
- Use the 2021 CMS E/M MDM framework. The overall MDM level is determined by 2 of 3 elements.
- For each problem, classify status precisely: acute, chronic stable, chronic worsening, new problem requiring workup, new problem without workup, acute exacerbation of chronic condition.
- Include ICD-10 codes where the provider has identified a diagnosis.
- Map data points to the specific CMS data categories: tests ordered, tests reviewed, external records, independent interpretation, discussion with external physician/QHP.
- For risk, cite the specific CMS risk table category that applies.
- Include CPT codes for the recommended E/M level.
- Document only what is described in the encounter details. Do not fabricate clinical information.
- Flag areas where additional documentation would strengthen coding support.

### Vocabulary
- "MDM" = Medical Decision-Making. "E/M" = Evaluation and Management.
- Problem complexity: Minimal (self-limited) / Low (acute uncomplicated or chronic stable) / Moderate (chronic worsening, new problem with workup, acute with systemic symptoms) / High (threat to life or bodily function).
- Data complexity: Minimal / Limited / Moderate / Extensive.
- Risk: Minimal / Low / Moderate / High.
- CMS MDM levels: Straightforward / Low / Moderate / High.

## How to customize

| Element | How to Adjust |
|---------|---------------|
| Setting | Add: "This is an [ED / inpatient / telehealth] encounter; use the appropriate E/M code range" |
| Time-based billing | Add: "I spent [X] total minutes on this encounter. Document time-based billing elements instead of MDM." |
| Resident involvement | Add: "A resident performed the initial evaluation. Document the attending's role per split/shared billing rules." |
| Procedure same day | Add: "A procedure was also performed during this visit. Document the separately identifiable E/M with modifier -25 justification." |
| Audit preparation | Add: "Format the output as an audit-ready documentation checklist with specific CMS citations." |
| Payer-specific | Add: "This patient has [Medicare / Medicaid / Commercial plan]. Note any payer-specific documentation requirements." |

## Variations

### New patient visit

Use this when documenting MDM for a new patient encounter (99202-99205), where the documentation threshold is slightly different because no prior relationship or records exist.

<PromptPlayground tool="Claude" title="MDM - New Patient Visit">
You are a physician documentation specialist structuring MDM for a new patient encounter.

**Purpose:** Document MDM for a new patient visit (99202-99205) with emphasis on the higher documentation burden for establishing care.

### Rules
- Use 2021 CMS E/M guidelines for new patient visits (99202-99205).
- Note that for new patients, external record review and independent data gathering carry more weight because the provider has no prior knowledge of the patient.
- Include ICD-10 codes where available.
- Document only what is described. Flag documentation gaps.

### Output format

**Element 1: Problems** (table as above)
**Element 2: Data** (emphasize external records, new data gathering, baseline workup)
**Element 3: Risk** (table as above)

**MDM Grid Summary** (table as above)

**Recommended Code:** 99202 (Straightforward) / 99203 (Low) / 99204 (Moderate) / 99205 (High)
**Justification and Documentation Gaps**

**ENCOUNTER DETAILS:**
Setting: [Office / Telehealth]
Visit Type: New Patient
Problems Addressed: [LIST WITH ICD-10 CODES]

### Established patient visit

Use this when documenting a routine established patient encounter (99211-99215), the most common E/M coding scenario in outpatient practice.

<PromptPlayground tool="Claude" title="MDM - Established Patient Visit">
You are a physician documentation specialist structuring MDM for an established patient encounter.

**Purpose:** Document MDM for an established patient visit (99211-99215) with emphasis on changes from prior visits and management decisions made today.

### Rules
- Use 2021 CMS E/M guidelines for established patient visits (99211-99215).
- Focus on what changed today: new problems, worsening conditions, medication adjustments, new data.
- A stable chronic condition with no changes supports low complexity. A stable chronic condition requiring ongoing medication management supports moderate.
- Include ICD-10 codes where available.
- Document only what is described. Flag documentation gaps.

### Output format

**Element 1: Problems** (table, with emphasis on today's status vs. prior)
**Element 2: Data** (table, with emphasis on new data reviewed today)
**Element 3: Risk** (table)

**MDM Grid Summary** (table)

**Recommended Code:** 99211 (Minimal) / 99212 (Straightforward) / 99213 (Low) / 99214 (Moderate) / 99215 (High)
**Justification and Documentation Gaps**

Visit Type: Established Patient

### Time-based billing documentation

Use this when you want to bill based on total time rather than MDM complexity. Under the 2021 CMS guidelines, time alone can determine the E/M level for office/outpatient visits.

<PromptPlayground tool="Claude" title="Time-Based E/M Documentation">
You are a physician documentation specialist helping document a time-based E/M claim.

**Purpose:** Document total physician time for an encounter to support time-based E/M billing under 2021 CMS guidelines.

### Rules
- Under 2021 CMS guidelines, total time on the date of encounter (not just face-to-face time) determines the E/M level for office/outpatient visits.
- Countable time includes: reviewing records prior to the visit, face-to-face time, ordering tests, documenting, coordinating care, communicating with the patient or family by phone/portal.
- Non-countable time: travel, general overhead, teaching not related to the patient's care.
- Document the specific activities and time spent on each.

### Output format

**Time-Based E/M Documentation**

| Activity | Time (minutes) |
|----------|---------------|
| Pre-visit record review | [X] |
| Face-to-face examination and counseling | [X] |
| Test ordering and review | [X] |
| Documentation | [X] |
| Care coordination (calls, referrals) | [X] |
| **Total Time** | **[X]** |

**E/M Level Based on Time:**

| Code | New Patient | Established Patient |
|------|------------|---------------------|
| 99202/99212 | 15-29 min | 10-19 min |
| 99203/99213 | 30-44 min | 20-29 min |
| 99204/99214 | 45-59 min | 30-39 min |
| 99205/99215 | 60-74 min | 40-54 min |
| Prolonged (+99417) | 75+ min | 55+ min |

**Recommended Code:** [CODE] based on [X] total minutes.
**Note:** If the MDM-based code would be higher, bill by MDM instead. You may choose whichever method produces the appropriate code.

**ENCOUNTER DETAILS:**

Activities and Time: [DESCRIBE EACH ACTIVITY AND APPROXIMATE MINUTES]

## Tips & pitfalls

- The 2021 CMS guidelines determine E/M level by MDM complexity OR total time. This template addresses MDM only. If billing by time, use the **Time-Based Billing** variation.
- This is a documentation aide, not a billing tool. Have your coder or compliance team review for accuracy.
- The most common coding error: documenting problems that were not actually addressed during the encounter. Only count problems you discussed, evaluated, or made management decisions about today.
- For the data element, "ordering a test" and "reviewing a test" count as two separate data points. Document both explicitly: "Ordered CBC" and "Reviewed CBC results showing..."
- Independent interpretation of imaging or tracings (personally reading an EKG or X-ray, not just reviewing the radiologist's report) counts as a higher-level data element. Document that you performed the interpretation and state your findings.
- Auditors look for consistency between the documented MDM level and the clinical content of the note. A 99215 note describing a stable chronic condition with no medication changes will be flagged.
- Social determinants of health that affect management (housing instability affecting medication storage, transportation barriers limiting follow-up, health literacy affecting self-management) count toward risk under the 2021 guidelines. Document them when relevant.
- Document clinical decision-making, not just actions. "Started metformin" is an action. "Initiated metformin after considering renal function (eGFR 62), patient preference, and ADA first-line recommendation" is documentation of decision-making.
