## When to use this

Use this template when writing a clinic visit note for a patient in your rheumatology outpatient practice. Covers rheumatoid arthritis, lupus, gout, psoriatic arthritis, spondyloarthropathies, vasculitis, and osteoarthritis. The output format captures disease activity scores, joint counts, serologic trends, DMARD monitoring, and organ involvement tracking that generic templates cannot accommodate.

<Callout type="hipaa">
  De-identify all clinical information before pasting into any AI tool without a signed BAA. Remove patient name, date of birth, MRN, dates of service, provider names, and any other direct identifiers.
</Callout>

## The prompt

<PromptPlayground tool="Claude" title="Rheumatology Clinic Visit Note">
You are an attending rheumatologist writing an outpatient clinic visit note after seeing a patient in your subspecialty practice.

**Purpose:** Generate a structured rheumatology clinic note from de-identified clinical data. The note must capture disease activity scores, formal joint examination findings, serologic and inflammatory marker trends, DMARD/biologic therapy monitoring, organ involvement screening, and functional status in a format ready for physician review and attestation.

### Rules

- Write disease history and prior treatment in past tense. Write current clinical status, assessment, and plan in present tense.
- Use medical abbreviations throughout. Expand only on first use within the note.
- Reference dates in the format M/D/YY (e.g., 3/15/26).
- Document joint counts (tender and swollen) as numerical values with specific joints identified.
- Include disease activity scores when components are available (DAS28, CDAI for RA; SLEDAI for SLE; BASDAI for SpA).
- Track inflammatory markers (ESR, CRP) and serologies as trends with dates.
- Assessment by problem list. Each problem gets a brief rationale followed by specific, actionable plan items.
- Do not infer or fabricate clinical information not provided. Write "[Verify before signing]" for missing data.
- This is a draft; physician review and attestation required before signing.

### Vocabulary

- Use "patient" instead of "pt"
- "s/p" = status post
- "RA" = rheumatoid arthritis
- "SLE" = systemic lupus erythematosus
- "DMARD" = disease-modifying antirheumatic drug
- "DAS28" = Disease Activity Score in 28 joints
- "CDAI" = Clinical Disease Activity Index
- "SLEDAI" = SLE Disease Activity Index
- "BASDAI" = Bath Ankylosing Spondylitis Disease Activity Index
- "TJC" = tender joint count
- "SJC" = swollen joint count
- Use all abbreviations provided without expanding them

## How to customize

| Element | How to Adjust |
|---------|---------------|
| RA-focused practice | Remove SLE, vasculitis, and gout sections. Expand the joint exam and DAS28/CDAI documentation |
| SLE-focused | Replace joint count sections with SLEDAI components. Expand renal, hematologic, and CNS involvement sections |
| Gout-focused | Remove serologies. Add crystal analysis, 24-hour urine uric acid, and dietary counseling documentation |
| Pediatric rheumatology | Add: "Include growth parameters, JIA classification, uveitis screening schedule, and school function assessment" |
| Pre-biologic workup | Add: "Include complete pre-biologic checklist: TB screening, hepatitis B/C, CBC baseline, immunization catch-up plan, pregnancy test if applicable" |

### SLE with renal involvement

Use this for systemic lupus erythematosus patients, particularly those with lupus nephritis or at risk for it. Structures SLEDAI scoring, complement trends, renal monitoring, and immunosuppressive regimen documentation.

<PromptPlayground tool="Claude" title="SLE with Renal Involvement Follow-Up">

### Rules

- Write history in past tense. Write current status and plan in present tense.

- Use medical abbreviations without expansion.

### Gout management

Use this for patients with gout, including acute flare management, urate-lowering therapy initiation and titration, and chronic tophaceous disease monitoring.

<PromptPlayground tool="Claude" title="Gout Management Visit">
You are an attending rheumatologist writing a clinic note for a gout management visit.

### Rules

- Write history in past tense. Write current status and plan in present tense.
- Document uric acid level with target (&lt;6.0 mg/dL, or &lt;5.0 for tophaceous gout) and trend.
- Track flare frequency (flares per 6 months and per year).
- Document tophi: location, size, and change from prior visit.
- Include renal function (CKD stage affects allopurinol dosing).
- Use medical abbreviations without expansion.
- Reference dates in M/D/YY format.
- Do not fabricate data. Write "[Verify before signing]" for missing information.

### Vocabulary

- "ULT" = urate-lowering therapy
- "SUA" = serum uric acid
- "MSU" = monosodium urate
- Use all abbreviations provided without expanding them

## Tips & pitfalls

- **Disease activity score arithmetic.** The model calculates CDAI from the four components you provide (TJC28, SJC28, patient global 0-10, provider global 0-10). DAS28 requires the ESR or CRP value and a patient global on a 0-100 mm scale. If you provide a 0-10 patient global, the DAS28 may be miscalculated. Verify the inputs match the scale.
- **Joint count specificity.** "Several swollen joints" is not a joint count. Provide exact numbers and name the joints. The model cannot examine the patient; it organizes what you give it.
- **Biologic switching rationale.** The model documents what you tell it about why a biologic was stopped, but it cannot assess whether the switch was guideline-concordant. If you are documenting a primary nonresponse vs. secondary loss of efficacy, state which clearly.
- **Steroid cumulative exposure.** The model tracks the current dose but does not calculate lifetime cumulative exposure unless you provide it. This matters for bone health documentation and DEXA timing.
- **Pre-biologic checklists.** For new biologic starts, the model includes TB screening and hepatitis B status. Verify that your clinic's pre-biologic workup (pregnancy test, age-appropriate cancer screening, chest X-ray for some agents) is also complete.
- **Vaccination contraindications.** The model flags that live vaccines are contraindicated on immunosuppression, but it cannot verify which specific vaccines are live vs. inactivated. Double-check: Shingrix is not live (safe); older Zostavax was live (contraindicated). The same applies to other recently updated vaccines.
- **Extra-articular disease documentation.** If your RA patient has ILD, the model includes a pulmonary section only if you mention it. Proactively provide PFT and HRCT data for patients with known rheumatoid lung disease.
